## Tujuan

AI memastikan project memenuhi standar production-ready.

## Checklist Production

1. Auth aman.
2. Role access benar.
3. Database terhubung.
4. Error handling ada.
5. Loading state ada.
6. Empty state ada.
7. UI responsive.
8. Build berhasil.
9. Env production siap.
10. Tidak ada secret bocor.
11. Tidak ada console log berlebihan.
12. Tidak ada data dummy di halaman production utama.
13. Tidak ada link localhost di production.
14. Tidak ada halaman mati.
15. Semua CTA bekerja.

## LMS Production Standard

LMS dianggap siap pakai jika:

* Admin bisa mengelola sistem.
* Mentor bisa membuat/mengupload materi.
* Peserta bisa belajar.
* Progress tersimpan.
* Sertifikat muncul otomatis.
* Dashboard tiap role jelas.
* Deployment stabil.
* Database tidak mudah rusak.
* UI tidak membingungkan.

## Prinsip Penyempurnaan Otomatis

Jika user meminta “sempurnakan website”, AI harus:

1. Audit semua lini.
2. Perbaiki bug prioritas.
3. Lengkapi fitur inti.
4. Rapikan UI/UX.
5. Perkuat auth dan role.
6. Hubungkan data.
7. Siapkan deployment.
8. Dokumentasikan hasil.

## Final Review

Sebelum selesai, AI harus menjawab:

* Apakah fitur utama berjalan?
* Apakah semua role punya alur jelas?
* Apakah ada error build?
* Apakah ada env yang kurang?
* Apakah ada risiko keamanan?
* Apakah UI siap dipakai pengguna?
